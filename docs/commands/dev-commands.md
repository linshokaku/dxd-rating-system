# 開発者向けコマンド

## ダミーユーザー共通ルール

- ダミーユーザーとして扱う `discord_user_id` は `1` 以上 `1000` 以下とする。
- `dev_register` では、この範囲外の `discord_user_id` は不正として扱う。
- `dev_join`、`dev_present`、`dev_leave` で対象ユーザーがダミーユーザーである場合、後続通知メッセージの先頭は Discord mention ではなく `<dummy_{dummy_user_id}> ` の形式とする。

## `/dev_register`

### 目的
`/register` の動作確認用に、開発者が任意の Discord user ID をダミープレイヤーとして登録する。

### 実行者
- `admin` 権限を持つユーザーのみ。

### 入力
- `discord_user_id` (必須): 追加したいダミーユーザーの Discord user ID。

### 正常時の挙動
- 指定した `discord_user_id` のプレイヤーをマッチングシステムに登録する。
- `discord_user_id` はダミーユーザー用の ID として `1` 以上 `1000` 以下のみを受け付ける。
- レスポンスを返す
  - `ダミーユーザーを登録しました。`

### エラー時の挙動
- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `discord_user_id` がすでに登録済みの場合:
  - `指定したユーザーはすでに登録済みです。`
- `discord_user_id` の形式が不正な場合:
  - `discord_user_id が不正です。`
- `discord_user_id` が `1` 以上 `1000` 以下の範囲外である場合:
  - `discord_user_id が不正です。`
- 内部エラーが発生した場合:
  - `ダミーユーザーの登録に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- `admin` 限定。

### 備考
- 開発・検証用途専用コマンドとして扱う。
- 同じ `discord_user_id` の重複登録は行わない。
- 作成されるデータ形式は `/register` で作成されるプレイヤーと同一とする。
- ダミーユーザーとして登録できる `discord_user_id` は `1` 以上 `1000` 以下に限定する。

## `/dev_join`

### 目的
`/join` の動作確認用に、開発者が任意の Discord user ID をマッチングキューへ参加させる。

### 実行者
- `admin` 権限を持つユーザーのみ。

### 入力
- `discord_user_id` (必須): キューへ参加させたい Discord user ID。
- `match_format` (必須): 参加させたいフォーマット。
- `queue_name` (必須): 参加させたいキュー名。

初期状態では、`match_format` は以下を受け付ける。

- `1v1`
- `2v2`
- `3v3`

初期状態では、各フォーマットについて `queue_name` は以下を受け付ける。

- `low`
- `high`

### 正常時の挙動
- 指定した `discord_user_id` に対応するプレイヤーを、指定した `match_format` と `queue_name` のマッチングキューへ参加させる。
- `match_format` と `queue_name` は内部で `queue_class_id` に解決して扱う。
- 参加条件がある場合は、指定した `match_format` のレーティングで参加可否を判定する。
- 同時に参加できるキューは、全フォーマット全階級を通して 1 つだけとする。
- レスポンスを返す
  - `指定したユーザーをキューに参加させました。`
- `join` をきっかけにマッチ成立した場合でも、まずは `dev_join` のレスポンスを返し、その後の別通知で `マッチ成立` を伝える。

### エラー時の挙動
- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `discord_user_id` の形式が不正な場合:
  - `discord_user_id が不正です。`
- 指定した `match_format` が無効な場合:
  - `指定したフォーマットは存在しません。`
- 指定した `queue_name` が無効な場合:
  - `指定したキューは存在しません。`
- 指定した `discord_user_id` が未登録の場合:
  - `指定したユーザーは未登録です。`
- 指定したユーザーが、実行時点の指定フォーマットのレーティングではそのキューに参加できない場合:
  - `指定したユーザーは現在のレーティングではそのキューに参加できません。`
- 指定したユーザーがすでにキュー参加中の場合:
  - `指定したユーザーはすでにキュー参加中です。`
- 内部エラーが発生した場合:
  - `ダミーユーザーのキュー参加に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- `admin` 限定。

### 備考
- 開発・検証用途専用コマンドとして扱う。
- 後続通知の送信先は、このコマンドを実行した channel を基準に決定する想定。
- 対象ユーザーがダミーユーザーである場合、後続通知メッセージの先頭は `<dummy_{dummy_user_id}> ` とする。
- 参加条件の判定は `dev_join` 実行時にのみ行い、待機中の再判定は行わない。
- 詳細仕様は `docs/matching/common.md` と `docs/matching/queue_classes.md` を参照する。

## `/dev_present`

### 目的
`/present` の動作確認用に、開発者が任意の Discord user ID の在席を更新する。

### 実行者
- `admin` 権限を持つユーザーのみ。

### 入力
- `discord_user_id` (必須): 在席を更新したい Discord user ID。

### 正常時の挙動
- 指定した `discord_user_id` に対応する、現在参加中の `waiting` 状態のキュー行を更新する。
- レスポンスを返す
  - `指定したユーザーの在席を更新しました。`

### エラー時の挙動
- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `discord_user_id` の形式が不正な場合:
  - `discord_user_id が不正です。`
- 指定した `discord_user_id` が未登録の場合:
  - `指定したユーザーは未登録です。`
- 指定したユーザーがキューに参加していない場合:
  - `指定したユーザーはキューに参加していません。`
- 実行時点で指定したユーザーが期限切れだった場合:
  - `指定したユーザーは期限切れのためキューから外れました。`
- 内部エラーが発生した場合:
  - `ダミーユーザーの在席更新に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- `admin` 限定。

### 備考
- 開発・検証用途専用コマンドとして扱う。
- 後続通知の送信先は、このコマンド実行時の channel 情報で上書きされる想定。
- 対象ユーザーがダミーユーザーである場合、後続通知メッセージの先頭は `<dummy_{dummy_user_id}> ` とする。
- `match_format` や `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。

## `/dev_leave`

### 目的
`/leave` の動作確認用に、開発者が任意の Discord user ID をマッチングキューから退出させる。

### 実行者
- `admin` 権限を持つユーザーのみ。

### 入力
- `discord_user_id` (必須): キューから退出させたい Discord user ID。

### 正常時の挙動
- 指定した `discord_user_id` に対応する、現在参加中の `waiting` 状態のキュー行がある場合は、キューから退出させる。
- `waiting` 行がない場合も冪等に成功扱いとする。
- レスポンスを返す
  - `指定したユーザーをキューから退出させました。`

### エラー時の挙動
- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `discord_user_id` の形式が不正な場合:
  - `discord_user_id が不正です。`
- 指定した `discord_user_id` が未登録の場合:
  - `指定したユーザーは未登録です。`
- 実行時点で指定したユーザーが期限切れだった場合:
  - `指定したユーザーはすでに期限切れでキューから外れています。`
- 内部エラーが発生した場合:
  - `ダミーユーザーのキュー退出に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- `admin` 限定。

### 備考
- 開発・検証用途専用コマンドとして扱う。
- 成功時はローカルで保持している在席確認リマインドタスクと期限切れタスクを cancel する想定。
- 対象ユーザーがダミーユーザーである場合、後続通知メッセージの先頭は `<dummy_{dummy_user_id}> ` とする。
- `match_format` や `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。

## `/dev_is_admin`

### 目的

実行した人が admin ユーザーかどうかを答える。

### 実行者
- サーバー内の全ての Discord ユーザー。

### 入力
- なし。


### 正常時の挙動
- 実行したユーザーが admin ユーザーの場合
    - "はい" とレスポンスする
- 実行したユーザーが admin ユーザーでない場合
    - "いいえ" とレスポンスする

### エラー時の挙動
- 内部エラーが発生した場合:
    - `エラーが発生しました。管理者に確認してください。`
    - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。
