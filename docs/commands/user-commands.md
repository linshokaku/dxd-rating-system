# ユーザー向けコマンド

## `/register`

### 目的
実行者をマッチングシステムのプレイヤーとして登録する。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーの Discord user ID をマッチングシステムに登録する。
- レスポンスを返す
  - `登録が完了しました。`

### エラー時の挙動
- すでに登録済みの場合:
  - `すでに登録済みです。`
- 内部エラーが発生した場合:
  - `登録に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

## `/join`

### 目的
実行者をマッチングキューに参加させる。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- `match_format` (必須): 参加したいフォーマット。
- `queue_name` (必須): 参加したいキュー名。

初期状態では、`match_format` は以下を受け付ける。

- `1v1`
- `2v2`
- `3v3`

初期状態では、各フォーマットについて `queue_name` は以下を受け付ける。

- `low`
- `high`

### 正常時の挙動
- 実行したユーザーに対応するプレイヤーを、指定した `match_format` と `queue_name` のマッチングキューへ参加させる。
- `match_format` と `queue_name` は内部で `queue_class_id` に解決して扱う。
- 参加条件がある場合は、指定した `match_format` の稼働中シーズンのレーティングで参加可否を判定する。
- そのレート行が carryover 未確定なら、判定前に carryover を確定する。
- 同時に参加できるキューは、全フォーマット全階級を通して 1 つだけとする。
- レスポンスを返す
  - `キューに参加しました。5分間マッチングします。`
- `join` をきっかけにマッチ成立した場合でも、まずは `join` のレスポンスを返し、その後の別通知で `マッチ成立` を伝える。

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 指定した `match_format` が無効な場合:
  - `指定したフォーマットは存在しません。`
- 指定した `queue_name` が無効な場合:
  - `指定したキューは存在しません。`
- 実行時点の指定フォーマットのレーティングでは指定キューの参加条件を満たさない場合:
  - `現在のレーティングではそのキューに参加できません。現在レート: 1487`
- すでにキュー参加中の場合:
  - `すでにキュー参加中です。`
- 内部エラーが発生した場合:
  - `キュー参加に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時は在席確認リマインドと期限切れ処理の対象になる。
- 後続通知の送信先は、このコマンドを実行した channel を基準に決定する想定。
- 参加条件の判定は `join` 時にのみ行い、待機中の再判定は行わない。
- シーズン切替時にも既存の待機行はそのまま残り、参加条件は再判定しない。
- 詳細仕様は `docs/matching/common.md` と `docs/matching/queue_classes.md` を参照する。

## `/present`

### 目的
実行者の在席を更新し、マッチングキューの期限を延長する。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーに対応する、現在参加中の `waiting` 状態のキュー行を更新する。
- `last_present_at` と `expire_at` を更新し、期限を 5 分延長する。
- レスポンスを返す
  - `在席を更新しました。次の期限は5分後です。`

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- キューに参加していない場合:
  - `キューに参加していません。`
- 実行時点で期限切れだった場合:
  - `期限切れのためキューから外れました。`
- 内部エラーが発生した場合:
  - `在席更新に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時は在席確認リマインドと期限切れ処理のタイマーを新しい期限で張り直す。
- 後続通知の送信先は、このコマンド実行時の channel 情報で上書きされる想定。
- `match_format` や `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。

## `/leave`

### 目的
実行者をマッチングキューから退出させる。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーに対応する、現在参加中の `waiting` 状態のキュー行がある場合は、キューから退出させる。
- `waiting` 行がない場合も冪等に成功扱いとする。
- レスポンスを返す
  - `キューから退出しました。`

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 実行時点で期限切れだった場合:
  - `すでに期限切れでキューから外れています。`
- 内部エラーが発生した場合:
  - `キュー退出に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時はローカルで保持している在席確認リマインドタスクと期限切れタスクを cancel する想定。
- `match_format` や `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。

## `/match_spectate`

### 目的
実行者を、運営中の試合の観戦者として登録する。

### 実行者
`/register` 済みの Discord ユーザー。

### 入力
- `match_id` (必須): 観戦応募したい試合の `match_id`。

### 正常時の挙動
- 実行者を、指定した試合の観戦者として登録する。
- 観戦枠上限は `12 - 試合の参加人数` で判定する。
- 同一試合への重複応募は受け付けない。
- レスポンスを返す
  - `観戦応募を受け付けました。現在 3 / 6 人です。`

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 指定した試合が存在しない場合:
  - `指定した試合が見つかりません。`
- 指定した試合が観戦受付中でない場合:
  - `この試合は観戦受付を終了しています。`
- 実行者がその試合の参加者である場合:
  - `この試合の参加者は観戦応募できません。`
- すでに応募済みの場合:
  - `すでにこの試合へ観戦応募済みです。`
- 観戦枠が埋まっている場合:
  - `この試合の観戦枠は埋まっています。`
- 内部エラーが発生した場合:
  - `観戦応募に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 詳細仕様は `docs/matches/spectating.md` を参照する。
