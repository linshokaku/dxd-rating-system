# ユーザー向けコマンド

## `/register`

### 目的
実行者をマッチングシステムのプレイヤーとして登録する。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーの Discord user ID をマッチングシステムに登録する。
- レスポンスを返す
  - `登録が完了しました。`

### エラー時の挙動
- すでに登録済みの場合:
  - `すでに登録済みです。`
- 内部エラーが発生した場合:
  - `登録に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

## `/join`

### 目的
実行者をマッチングキューに参加させる。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- `queue_name` (必須): 参加したいキュー名。

初期状態では、以下を受け付ける。

- `low`
- `high`

### 正常時の挙動
- 実行したユーザーに対応するプレイヤーを、指定した `queue_name` のマッチングキューへ参加させる。
- `queue_name` は内部で `queue_class_id` に解決して扱う。
- 参加条件がある場合は、実行時点のレーティングで参加可否を判定する。
- レスポンスを返す
  - `キューに参加しました。5分間マッチングします。`
- `join` をきっかけにマッチ成立した場合でも、まずは `join` のレスポンスを返し、その後の別通知で `マッチ成立` を伝える。

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 指定した `queue_name` が無効な場合:
  - `指定したキューは存在しません。`
- 実行時点のレーティングでは指定キューの参加条件を満たさない場合:
  - `現在のレーティングではそのキューに参加できません。`
- すでにキュー参加中の場合:
  - `すでにキュー参加中です。`
- 内部エラーが発生した場合:
  - `キュー参加に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時は在席確認リマインドと期限切れ処理の対象になる。
- 後続通知の送信先は、このコマンドを実行した channel を基準に決定する想定。
- 参加条件の判定は `join` 時にのみ行い、待機中の再判定は行わない。

## `/present`

### 目的
実行者の在席を更新し、マッチングキューの期限を延長する。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーに対応する、現在参加中の `waiting` 状態のキュー行を更新する。
- `last_present_at` と `expire_at` を更新し、期限を 5 分延長する。
- レスポンスを返す
  - `在席を更新しました。次の期限は5分後です。`

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- キューに参加していない場合:
  - `キューに参加していません。`
- 実行時点で期限切れだった場合:
  - `期限切れのためキューから外れました。`
- 内部エラーが発生した場合:
  - `在席更新に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時は在席確認リマインドと期限切れ処理のタイマーを新しい期限で張り直す。
- 後続通知の送信先は、このコマンド実行時の channel 情報で上書きされる想定。
- `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。

## `/leave`

### 目的
実行者をマッチングキューから退出させる。

### 実行者
サーバー内の全ての Discord ユーザー。

### 入力
- なし。

### 正常時の挙動
- 実行したユーザーに対応する、現在参加中の `waiting` 状態のキュー行がある場合は、キューから退出させる。
- `waiting` 行がない場合も冪等に成功扱いとする。
- レスポンスを返す
  - `キューから退出しました。`

### エラー時の挙動
- プレイヤー未登録の場合:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 実行時点で期限切れだった場合:
  - `すでに期限切れでキューから外れています。`
- 内部エラーが発生した場合:
  - `キュー退出に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 権限
- 特になし。

### 備考
- 成功時はローカルで保持している在席確認リマインドタスクと期限切れタスクを cancel する想定。
- `queue_name` の入力は取らず、現在参加中のキューへ暗黙適用する。
