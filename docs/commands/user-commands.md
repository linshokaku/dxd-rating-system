# ユーザー向けコマンド

## 共通ルール

- `/register` は未登録ユーザーでも実行できる。
- `/register` 以外のコマンドはプレイヤー登録済みであることが前提。
- 未登録のまま対象コマンドを実行すると、`プレイヤー登録が必要です。先に /register を実行してください。` を返す。
- slash command の実行結果はプレーンテキストで返す。
- プレイヤー操作コマンドカテゴリに属するコマンドのレスポンスは、成功時も失敗時も interaction 宛てに、実行ユーザーにだけ見える形式で返す。
- 内部エラー時はコマンドごとの失敗メッセージを返し、例外詳細はそのまま Discord に出さない。

## コマンドカテゴリー

- プレイヤー操作コマンド:
  - `/register`
  - `/join`
  - `/present`
  - `/leave`
  - `/player_info`
  - `/player_info_season`
- 試合操作コマンド:
  - `/match_parent`
  - `/match_spectate`
  - `/match_win`
  - `/match_lose`
  - `/match_draw`
  - `/match_void`
  - `/match_approve`

## プレイヤー操作コマンド

プレイヤー登録、マッチングキュー操作、自分の成績確認を行うコマンド群とする。
このカテゴリのレスポンスは、成功時も失敗時も interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。

### `/register`

#### 目的

実行者をプレイヤーとして登録する。

#### 入力

- なし。

#### 正常時の挙動

- 実行者の Discord user ID をプレイヤーとして登録する。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `登録が完了しました。`

#### エラー時の挙動

- すでに登録済み:
  - interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
  - `すでに登録済みです。`
- 内部エラー:
  - interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
  - `登録に失敗しました。管理者に確認してください。`

#### 備考

- 公開チャンネルに設置する登録 UI の仕様は [../ui/register.md](../ui/register.md) を参照する。

### `/join`

#### 目的

マッチングキューに参加する。

#### 入力

- `match_format` (必須): 参加したいフォーマット。
- `queue_name` (必須): 参加したいキュー名。

現在の選択肢:

- `match_format`: `1v1`, `2v2`, `3v3`
- `queue_name`: `high`, `low`

#### 正常時の挙動

- 実行者を指定したマッチングキューに参加させる。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `キューに参加しました。5分間マッチングします。`

#### エラー時の挙動

- `match_format` が不正:
  - `指定したフォーマットは存在しません。`
- `queue_name` が不正:
  - `指定したキューは存在しません。`
- 現在のレーティングではそのキューに参加できない:
  - `現在のレーティングではそのキューに参加できません。`
- キュー参加が制限されている:
  - `現在キュー参加を制限されています。`
- すでにキュー参加中:
  - `すでにキュー参加中です。`
- 内部エラー:
  - `キュー参加に失敗しました。管理者に確認してください。`

#### 備考

- 有効な待機キューがすでにある場合は参加できない。
- このコマンドで作成される在席確認リマインドと期限切れ通知は、このコマンドを実行した channel に送る。
- 通知は対象ユーザーへの mention を先頭に付けた通常のテキストメッセージとして送る。
- その他の後続通知の送信先ポリシーは [../outbox.md](../outbox.md) を参照する。

### `/present`

#### 目的

在席を更新して、現在参加中のキューの期限を延長する。

#### 入力

- なし。

#### 正常時の挙動

- 現在参加中のキューの在席を更新する。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `在席を更新しました。次の期限は5分後です。`

#### エラー時の挙動

- キューに参加していない:
  - `キューに参加していません。`
- 実行時点で期限切れ:
  - `期限切れのためキューから外れました。`
- 内部エラー:
  - `在席更新に失敗しました。管理者に確認してください。`

#### 備考

- `match_format` や `queue_name` の入力は取らず、現在参加中のキューを対象にする。
- このコマンドで更新される在席確認リマインドと期限切れ通知の送信先は、このコマンドを実行した channel に上書きする。
- 通知は対象ユーザーへの mention を先頭に付けた通常のテキストメッセージとして送る。
- その他の後続通知の送信先ポリシーは [../outbox.md](../outbox.md) を参照する。

### `/leave`

#### 目的

現在参加中のマッチングキューから退出する。

#### 入力

- なし。

#### 正常時の挙動

- 現在参加中のキューから退出する。
- 参加中のキューがなくても成功扱いで返す。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `キューから退出しました。`

#### エラー時の挙動

- 実行時点で期限切れ:
  - `すでに期限切れでキューから外れています。`
- 内部エラー:
  - `キュー退出に失敗しました。管理者に確認してください。`

#### 備考

- `match_format` や `queue_name` の入力は取らず、現在参加中のキューを対象にする。

### `/player_info`

#### 目的

現在のシーズンにおける自分のプレイヤー情報を表示する。

#### 入力

- なし。

#### 正常時の挙動

- 現在のシーズンの成績をプレーンテキストで返す。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 返却形式:

```text
プレイヤー情報
1v1
rating: 1500.00
games_played: 0
wins: 0
losses: 0
draws: 0
last_played_at: -
2v2
...
3v3
...
```

#### エラー時の挙動

- 内部エラー:
  - `プレイヤー情報の取得に失敗しました。管理者に確認してください。`

#### 備考

- `last_played_at` は未対戦なら `-`、値がある場合は ISO 8601 形式で表示する。
- 現在はシーズン名や表示名ではなく、フォーマットごとの成績を返す。

### `/player_info_season`

#### 目的

指定したシーズンにおける自分のプレイヤー情報を表示する。

#### 入力

- `season_id` (必須): 表示したいシーズンの ID。

#### 正常時の挙動

- 指定したシーズンの成績をプレーンテキストで返す。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 返却形式:

```text
プレイヤー情報
season_id: 1
season_name: season 1
1v1
rating: 1500.00
games_played: 0
wins: 0
losses: 0
draws: 0
last_played_at: -
2v2
...
3v3
...
```

#### エラー時の挙動

- `season_id` に対応するシーズンが存在しない:
  - `指定したシーズンが見つかりません。`
- そのシーズンのプレイヤー情報が存在しない:
  - `指定したシーズンのプレイヤー情報はありません。`
- 内部エラー:
  - `シーズン別プレイヤー情報の取得に失敗しました。管理者に確認してください。`

## 試合操作コマンド

`/match_` prefix のコマンドで、試合への参加補助、観戦応募、勝敗報告、承認を行うコマンド群とする。

### 試合操作コマンド共通

- 対象コマンド:
  - `/match_parent`
  - `/match_spectate`
  - `/match_win`
  - `/match_lose`
  - `/match_draw`
  - `/match_void`
  - `/match_approve`
- いずれも `match_id` を引数に取る。
- 試合状態に起因するエラーでは、サービス側の文言をそのまま返す。
- 代表的な文言:
  - `指定した試合が見つかりません。`
  - `この試合の参加者ではありません。`
  - `この試合はすでに結果確定済みです。`
  - `この試合は観戦受付を終了しています。`
  - `まだ勝敗報告を受け付けていません。`
  - `この試合の勝敗報告は締め切られています。`
  - `承認期間中は勝敗報告を変更できません。`
  - `この試合は承認期間中ではありません。`
  - `この試合では承認は不要です。`

### `/match_parent`

#### 目的

試合の親に立候補する。

#### 入力

- `match_id` (必須): 対象の試合 ID。

#### 正常時の挙動

- 対象試合の親に立候補する。
- 成功時のレスポンス:
  - `親に立候補しました。`

#### エラー時の挙動

- 試合状態エラー:
  - `MatchFlowError` の文言をそのまま返す。
- 内部エラー:
  - `試合操作に失敗しました。管理者に確認してください。`

#### 備考

- 後続通知の送信先には、このコマンドを実行した channel 情報が使われる。

### `/match_spectate`

#### 目的

試合の観戦応募を行う。

#### 入力

- `match_id` (必須): 対象の試合 ID。

#### 正常時の挙動

- 対象試合の観戦者として登録する。
- 成功時のレスポンス:
  - `観戦応募を受け付けました。現在 {active_spectator_count} / {max_spectators} 人です。`

#### エラー時の挙動

- 観戦が制限されている:
  - `現在観戦を制限されています。`
- 試合状態エラー:
  - `MatchFlowError` の文言をそのまま返す。
- 内部エラー:
  - `観戦応募に失敗しました。管理者に確認してください。`
