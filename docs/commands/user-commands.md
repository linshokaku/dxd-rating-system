# ユーザー向けコマンド

## 共通ルール

- `/register` は未登録ユーザーでも実行できる。
- `/register` 以外のコマンドはプレイヤー登録済みであることが前提。
- 未登録のまま対象コマンドを実行すると、`プレイヤー登録が必要です。先に /register を実行してください。` を返す。
- slash command の実行結果はプレーンテキストで返す。
- プレイヤー操作コマンドカテゴリに属するコマンドのレスポンスは、成功時も失敗時も interaction 宛てに、実行ユーザーにだけ見える形式で返す。
- 内部エラー時はコマンドごとの失敗メッセージを返し、例外詳細はそのまま Discord に出さない。

## コマンドカテゴリー

- プレイヤー操作コマンド:
  - `/register`
  - `/join`
  - `/present`
  - `/leave`
  - `/player_info`
  - `/player_info_season`
  - `/info_thread`
  - `/leaderboard`
  - `/leaderboard_season`
- 試合操作コマンド:
  - `/match_parent`
  - `/match_spectate`
  - `/match_win`
  - `/match_lose`
  - `/match_draw`
  - `/match_void`
  - `/match_approve`

## プレイヤー操作コマンド

プレイヤー登録、マッチングキュー操作、自分の成績確認、ランキング確認を行うコマンド群とする。
このカテゴリのレスポンスは、成功時も失敗時も interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。

### `/register`

#### 目的

実行者をプレイヤーとして登録する。

#### 入力

- なし。

#### 正常時の挙動

- 実行者の Discord user ID をプレイヤーとして登録する。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `登録が完了しました。`

#### エラー時の挙動

- すでに登録済み:
  - interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
  - `すでに登録済みです。`
- 内部エラー:
  - interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
  - `登録に失敗しました。管理者に確認してください。`

#### 備考

- 公開チャンネルに設置する登録 UI の仕様は [../ui/register.md](../ui/register.md) を参照する。

### `/join`

#### 目的

マッチングキューに参加する。

#### 入力

- `match_format` (必須): 参加したいフォーマット。
- `queue_name` (必須): 参加したいキュー名。

現在の選択肢:

- `match_format`: `1v1`, `2v2`, `3v3`
- `queue_name`: `beginner`, `regular`, `master`

初期状態の参加条件:

- `beginner`: 対象フォーマットの現在レートが `1600` 未満
- `regular`: 対象フォーマットの現在レートに関係なく参加可能
- `master`: 対象フォーマットの現在レートが `1600` 以上

#### 正常時の挙動

- 実行者を指定したマッチングキューに参加させる。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `キューに参加しました。5分間マッチングします。`

#### エラー時の挙動

- `match_format` が不正:
  - `指定したフォーマットは存在しません。`
- `queue_name` が不正:
  - `指定したキューは存在しません。`
- 現在のレーティングではそのキューに参加できない:
  - `現在のレーティングではそのキューに参加できません。`
- キュー参加が制限されている:
  - `現在キュー参加を制限されています。`
- すでにキュー参加中:
  - `すでにキュー参加中です。`
- 内部エラー:
  - `キュー参加に失敗しました。管理者に確認してください。`

#### 備考

- 有効な待機キューがすでにある場合は参加できない。
- 成功時は、`レート戦マッチング` チャンネル配下に在席確認用の private thread を作成し、admin と実行者をその thread に入れる。
- 在席確認 thread の親チャンネルは、このコマンドをどのチャンネルで実行したかには依存しない。
- 在席確認、離席、残り 1 分リマインド、期限切れ通知は、その private thread に集約する。
- `レート戦マッチング` チャンネルに設置する UI は、このコマンドの代替導線として同じ業務処理と文言を使う。詳細は [../ui/matchmaking_channel.md](../ui/matchmaking_channel.md) を参照する。
- 在席確認 thread UI の詳細は [../ui/matchmaking_presence_thread.md](../ui/matchmaking_presence_thread.md) を参照する。
- その他の後続通知の送信先ポリシーは [../outbox.md](../outbox.md) を参照する。

### `/present`

#### 目的

在席を更新して、現在参加中のキューの期限を延長する。

#### 入力

- なし。

#### 正常時の挙動

- 現在参加中のキューの在席を更新する。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `在席を更新しました。次の期限は5分後です。`

#### エラー時の挙動

- キューに参加していない:
  - `キューに参加していません。`
- 実行時点で期限切れ:
  - `期限切れのためキューから外れました。`
- 内部エラー:
  - `在席更新に失敗しました。管理者に確認してください。`

#### 備考

- `match_format` や `queue_name` の入力は取らず、現在参加中のキューを対象にする。
- 在席確認 thread の `在席報告` ボタンでも同じ業務処理を実行する。
- 通常ユーザーの待機では、在席確認リマインドと期限切れ通知は参加時に作成された在席確認 thread に継続して送る。
- 在席確認 thread UI の詳細は [../ui/matchmaking_presence_thread.md](../ui/matchmaking_presence_thread.md) を参照する。
- その他の後続通知の送信先ポリシーは [../outbox.md](../outbox.md) を参照する。

#### 正常時の挙動

- 参加中のキューがなくても成功扱いで返す。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `キューから退出しました。`

#### エラー時の挙動

- 実行時点で期限切れ:
  - `すでに期限切れでキューから外れています。`
- 内部エラー:
  - `キュー退出に失敗しました。管理者に確認してください。`

#### 備考

- `match_format` や `queue_name` の入力は取らず、現在参加中のキューを対象にする。
- 在席確認 thread の `離席` ボタンでも同じ業務処理を実行する。
- 在席確認 thread UI の詳細は [../ui/matchmaking_presence_thread.md](../ui/matchmaking_presence_thread.md) を参照する。

### `/player_info`

#### 目的

現在のシーズンにおける自分のプレイヤー情報を表示する。

#### 入力

- なし。

#### 正常時の挙動

- 現在のシーズンの成績をプレーンテキストで、実行ユーザーに現在紐づいている情報確認用 private thread に送る。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- thread への投稿形式:

```text
プレイヤー情報
1v1
rating: 1500.00
games_played: 0
wins: 0
losses: 0
draws: 0
last_played_at: -
2v2
...
3v3
...
```

- 成功時のレスポンス:
  - `プレイヤー情報を表示しました。`

#### エラー時の挙動

- 実行ユーザーに紐づく情報確認用 thread が存在しない:
  - `先に /info_thread を実行してください。`
- 紐づけ先の thread が存在しない、または利用できない:
  - `情報確認用スレッドが見つかりません。先に /info_thread を実行してください。`
- 内部エラー:
  - `プレイヤー情報の取得に失敗しました。管理者に確認してください。`

#### 備考

- `last_played_at` は未対戦なら `-`、値がある場合は ISO 8601 形式で表示する。
- 現在はシーズン名や表示名ではなく、フォーマットごとの成績を返す。
- thread への表示形式は [../ui/info_thread.md](../ui/info_thread.md) を参照する。

- 成功時のレスポンス:

### `/info_thread`

#### 目的

プレイヤー情報とランキング確認結果を受け取るための private thread を作成する。

#### 入力

- `command_name` (必須): 作成したい情報確認 thread の用途。

現在の選択肢:

- `leaderboard`
- `leaderboard_season`
- `player_info`
- `player_info_season`

#### 正常時の挙動

- `レート戦情報` チャンネル配下に、情報確認用の private thread を 1 つ作成する。
- private thread の親チャンネルは、このコマンドをどのチャンネルで実行したかには依存しない。
- 作成した thread には、少なくとも実行ユーザー、admin、Bot を参加させる。
- `command_name` は、作成直後の案内文と、その thread に表示する操作 UI の種類を決める。
- `/info_thread` 実行直後には、指定した `command_name` 相当の本文データを自動では表示しない。
- Bot は、作成した `thread_id` を実行ユーザーの最新の情報確認 thread として一時的に紐づける。
- 実行ユーザーに既存の紐づけがあっても、新しい thread を作成して最新紐づけを上書きする。
- `command_name` ごとの別管理は行わず、実行ユーザーごとに最新 1 件だけを保持する。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `情報確認用スレッドを作成しました。`

#### エラー時の挙動

- 情報確認用チャンネルが見つからない:
  - `情報確認用チャンネルが見つかりません。管理者に確認してください。`
- 内部エラー:
  - `情報確認用スレッドの作成に失敗しました。管理者に確認してください。`

#### 備考

- `レート戦情報` チャンネルに設置する公開 button UI は、このコマンドの代替導線として同じ業務処理と文言を使う。詳細は [../ui/info_channel.md](../ui/info_channel.md) を参照する。
- 作成直後の案内文と thread の可視性ルールは [../ui/info_thread.md](../ui/info_thread.md) を参照する。
- 情報確認用チャンネルの用途と権限は [../ui/registered_channels.md](../ui/registered_channels.md) を参照する。

### `/leaderboard`

#### 目的

指定したフォーマットの現在シーズンランキングを表示する。

#### 入力

- `match_format` (必須): 表示したいフォーマット。
- `page` (必須): 表示したいページ番号。

現在の選択肢:

- `match_format`: `1v1`, `2v2`, `3v3`

#### 正常時の挙動

- 現在のシーズンにおける、指定 `match_format` のランキングを取得する。
- 1 ページあたり 20 件ずつ取得する。
- 表示内容は、実行ユーザーに現在紐づいている情報確認用 private thread に送る。
- thread に表示する項目は、現在順位、ユーザー名、小数点付きのレート値、`1日前` の順位差分、`3日前` の順位差分、`7日前` の順位差分とする。
- thread に投稿するランキングメッセージには、次ページが存在する場合だけ末尾に `次のページ` button を付ける。
- `次のページ` button を押したときは、同じ `match_format` の `/leaderboard <match_format> page:n+1` と同等の処理を行う。
- `次のページ` button を押した時点で、押下元メッセージ上の button は disabled にする。
- レスポンスは interaction 宛てに、実行ユーザーにだけ見えるプレーンテキストで返す。
- 成功時のレスポンス:
  - `ランキングを表示しました。`

#### エラー時の挙動

- `match_format` が不正:
  - `指定したフォーマットは存在しません。`
- `page` が `1` 未満:
  - `page は 1 以上で指定してください。`
- 実行ユーザーに紐づく情報確認用 thread が存在しない:
  - `先に /info_thread を実行してください。`
- 紐づけ先の thread が存在しない、または利用できない:
  - `情報確認用スレッドが見つかりません。先に /info_thread を実行してください。`
- 指定したページに表示対象が存在しない:
  - `指定したページにはランキングがありません。`
- 内部エラー:
  - `ランキングの取得に失敗しました。管理者に確認してください。`

#### 備考

- このコマンドは新しい private thread を作成せず、現在紐づいている thread を表示先として使う。
- `次のページ` button は、slash command `/leaderboard` を直接実行して thread に投稿された結果にも、thread 内 UI から実行して投稿された結果にも同じ条件で付く。
- `leaderboard` 用 info thread 初回メッセージの `ランキングを表示` button と、ランキング結果メッセージの `次のページ` button は、一度押すと押下元メッセージ上の component 全体が disabled になる。
- ランキングの並び順と順位差分の計算ルールは [../leaderboard/ranking.md](../leaderboard/ranking.md) を参照する。
- thread への表示形式は [../ui/info_thread.md](../ui/info_thread.md) を参照する。

## 試合操作コマンド

`/match_` prefix のコマンドで、試合への参加補助、観戦応募、勝敗報告、承認を行うコマンド群とする。

### 試合操作コマンド共通

- 対象コマンド:
  - `/match_parent`
  - `/match_spectate`
  - `/match_win`
  - `/match_lose`
  - `/match_draw`
  - `/match_void`
  - `/match_approve`
- いずれも `match_id` を引数に取る。
- 試合状態に起因するエラーでは、サービス側の文言をそのまま返す。
- 代表的な文言:
  - `指定した試合が見つかりません。`
  - `この試合の参加者ではありません。`
  - `この試合はすでに結果確定済みです。`
  - `この試合は観戦受付を終了しています。`
  - `まだ勝敗報告を受け付けていません。`
  - `この試合の勝敗報告は締め切られています。`
  - `承認期間中は勝敗報告を変更できません。`
  - `この試合は承認期間中ではありません。`
  - `この試合では承認は不要です。`

### `/match_parent`

#### 目的

試合の親に立候補する。

#### 入力

- `match_id` (必須): 対象の試合 ID。

#### 正常時の挙動

- 対象試合の親に立候補する。
- 成功時のレスポンス:
  - `親に立候補しました。`

#### エラー時の挙動

- 試合状態エラー:
