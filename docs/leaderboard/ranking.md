# ランキング仕様

## 目的

- read-only な Web サービスおよび Web UI から、現在のランキングを表示できるようにする
- `1日前`、`3日前`、`7日前` からの順位変化量を、実装を複雑にしすぎずに表示できるようにする

## スコープ

本仕様が扱うのは以下とする。

- 現在ランキング
- `1日前`、`3日前`、`7日前` の順位比較
- ランキング表示の決定的な並び順

本仕様では、以下は扱わない。

- HTTP API の具体設計
- Web UI の画面デザイン
- Discord 上でのランキング通知

## 基本方針

- ランキングは `match_format` ごとに独立して持つ
- 現在ランキングの真実のソースは、稼働中シーズンの `player_format_stats` とする
- 表示名は Bot が保持する `players` の表示名キャッシュを利用してよい
- ランキング表示のために Discord API を直接参照することは必須としない
- レート値は内部では浮動小数のまま扱い、表示時だけ丸めてよい

## 対象プレイヤー

ランキング対象に含めるのは、対象 `match_format` について以下を満たすプレイヤーのみとする。

- `games_played > 0`

意図:

- `/register` 直後の未対戦プレイヤーが、初期値 `1500` のままランキングを埋めることを避ける

## 現在ランキングの並び順

対象プレイヤーは、以下の順で並べる。

1. `rating` 降順
2. `games_played` 降順
3. `player_id` 昇順

意図:

- 並び順を決定的にし、snapshot 生成時と現在表示時で同じ規則を使えるようにする
- 同率時にも順位がぶれないようにする

## 順位の定義

- 順位は上記の並び順に対する一意の連番とする
- 実装上は `row_number` 相当の順位を採用する
- 同率のプレイヤー同士でも同順位にはしない

補足:

- `dense_rank` や `rank` は採用しない
- 順位変化量の計算を単純にすることを優先する

## 表示項目

現在ランキングでは、少なくとも以下を表示できるようにする。

- `rank`
- `display_name`
- `match_format`
- `rating`
- `games_played`
- `wins`
- `losses`
- `draws`

## 比較対象日

順位変化量の比較対象は、以下の 3 種のみとする。

- `1日前`
- `3日前`
- `7日前`

これらは、いずれも JST 日付ベースの snapshot を参照する。

具体的には:

- `1日前` は `snapshot_date = current_jst_date - 1 day`
- `3日前` は `snapshot_date = current_jst_date - 3 days`
- `7日前` は `snapshot_date = current_jst_date - 7 days`

補足:

- `24時間前` や `168時間前` のような時刻単位比較は行わない
- 日次 snapshot だけを使うことで、実装を単純に保つ
- 比較対象 snapshot の `season_id` が現在ランキングの `season_id` と一致しない場合、その比較は行わない

## 順位変化量

各比較対象日について、順位変化量は以下で定義する。

```text
rank_change = past_rank - current_rank
```

解釈:

- 正の値: 順位が上がった
- 0: 順位変化なし
- 負の値: 順位が下がった

例:

- 3日前が `10位`、現在が `7位` の場合は `+3`
- 7日前が `2位`、現在が `5位` の場合は `-3`

## 比較不能時の扱い

以下の場合、その期間の順位変化量は `NULL` 相当とし、表示側では `-` などで表現してよい。

- 対象日の snapshot が存在しない
- 対象日の snapshot にそのプレイヤーが含まれていない
- 対象日の snapshot は存在するが `season_id` が現在ランキングと一致しない

補足:

- 初登場プレイヤーに対して特別な `NEW` 表示は必須としない
- 必要になった場合のみ、UI 側の追加表現として扱う

## 参照先

- 現在ランキング: 稼働中シーズンの `player_format_stats`
- 過去比較: `leaderboard_snapshots`

snapshot の詳細仕様は [snapshots.md](snapshots.md) を参照する。
