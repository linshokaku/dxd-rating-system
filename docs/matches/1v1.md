# 1v1 試合運営仕様

## 固定パラメータ

- `match_format = '1v1'`
- `team_size = 1`
- `participant_count = 2`

## 共通仕様の適用

試合進行は [common.md](common.md) をそのまま適用する。

`1v1` での具体的な読み替えは以下の通り。

- 親募集対象は 2 人
- 親の自動決定も 2 人の中からランダムに行う
- 全参加者報告は 2 人報告を意味する
- 報告人数に関する admin 通知の人数条件は `team_size = 1` を使う

## admin 通知に関する補足

`1v1` では、人数条件による admin 通知は「0 人報告」の場合のみ発火する。  
片方だけが報告した場合は、人数条件ではなく「片方のチームからしか報告がない」条件で admin 通知対象になる。

## バッチとの関係

`1v1` は 1 回のマッチングで 2 試合同時に生成するが、生成後の試合進行は各 `match_id` ごとに完全に独立して行う。
