# Discord UI 共通仕様

## 目的

Discord サーバー内に設置する UI の共通方針を定義する。

## 対象

- 公開チャンネルまたは専用メッセージに設置される Bot UI。
- button と select menu を主対象とする。

## 対象外

- UI を設置するための管理者コマンド。
- slash command そのものの入出力仕様。

## 基本方針

- この Bot は 1 つの Discord サーバーでのみ運用する前提とする。
- UI は `ui_type` 単位で仕様化する。
- UI の仕様とは別に、チャンネル自体の用途や permission overwrite を横断的に定義した方が分かりやすい場合は、専用のチャンネル仕様を分けてよい。
- 各 UI 仕様では、少なくとも以下を定義する。
  - 目的
  - 設置先の前提
  - 表示要素
  - ユーザー操作
  - 操作後の業務処理
  - 返答の可視性
  - 正常系とエラー系の文言
- UI 設置コマンドは、個別 UI の詳細実装に依存しすぎず、`ui_type` を指定して対象チャンネルに配置する前提で設計する。
- UI の設置方法と UI 自体の振る舞いは別仕様に分ける。
- UI アクションに対する返答は、slash command の返答ルールに縛られない。必要に応じて ephemeral message などを利用してよい。
- ただし、業務処理を起動する button は slash command と同じ返答モデルを使う。
- 業務処理を起動する button は、押下直後に押下ユーザーへ ephemeral defer を返し、処理完了後に成否を押下ユーザーだけに見える followup で返す。
- select menu など最終実行トリガーでない component は、状態保持や表示切り替えだけを行ってよく、必ずしも最終結果メッセージを返さなくてよい。
- 複数 component を組み合わせる UI では、どの操作が最終実行トリガーなのかを明示する。
- 公開チャンネルに設置する UI は、操作のたびに不要な結果メッセージを公開チャンネルへ流しすぎない。
- 同じ UI が複数回押された場合でも、業務処理は可能な限り冪等に扱う。

## 関連仕様

- 登録済みユーザーが利用できるチャンネル構成は [registered_channels.md](registered_channels.md) を参照する。

## 今後の拡張方針

- 新しい UI を追加する場合は、まずこの配下に `ui_type` ごとの仕様ファイルを追加する。
- UI を置くチャンネルの役割や権限が増えた場合は、必要に応じてチャンネル仕様を追加または更新する。
- UI 設置コマンド側の仕様では、どの `ui_type` をどこに配置できるかを別途定義する。
