# マッチングチャンネル UI 仕様

## UI 識別子

- `ui_type`: `matchmaking_channel`

## 目的

登録済みユーザーが、`レート戦マッチング` チャンネルに設置された UI からマッチングキューへ参加できるようにする。

## 対象外

- マッチング UI を設置する管理者コマンドの仕様。
- 在席更新とキュー退出の UI 化。
- マッチ成立後に作成される private thread の詳細仕様。

## 前提

- この UI は、登録済みユーザー向けの `レート戦マッチング` チャンネルに設置する。
- チャンネル自体の用途と権限は [registered_channels.md](registered_channels.md) を参照する。
- UI 設置コマンドの仕様は [setup_channel.md](setup_channel.md) を参照する。
- この UI は `/join` の代替導線であり、業務処理、参加条件判定、成功文言、失敗文言は `/join` と同じものを使う。
- `/join` は引き続き有効な導線として残す。
- 現時点では、在席更新とキュー退出はこの UI に含めず、`/present` と `/leave` を使う。

## 公開 UI の表示要素

- マッチング案内用のメッセージを 1 つ設置する。
- メッセージには、少なくとも以下を表示する。
  - この UI がマッチングキュー参加用であること。
  - 試合形式と階級を選んでから参加ボタンを押すこと。
  - 在席更新とキュー退出は slash command で行うこと。
- メッセージには、以下の component を設置する。
  - 試合形式を選ぶプルダウン 1 つ
  - 階級を選ぶプルダウン 1 つ
  - 参加ボタン 1 つ
- 参加ボタン以外に、在席更新ボタンや退出ボタンは置かない。

## 選択肢

### 試合形式プルダウン

- `match_format` を選ぶための component とする。
- 現時点の選択肢は以下とする。
  - `1v1`
  - `2v2`
  - `3v3`

### 階級プルダウン

- `queue_name` を選ぶための component とする。
- 選択肢は、選ばれている `match_format` に対応する階級定義から組み立てる。
- 初期状態の選択肢は以下とする。
  - `beginner`
  - `regular`
  - `master`
- 初期状態の参加条件は以下とする。
  - `beginner`: 対象フォーマットの現在レートが `1600` 未満
  - `regular`: 対象フォーマットの現在レートに関係なく参加可能
  - `master`: 対象フォーマットの現在レートが `1600` 以上
- 将来フォーマットごとに階級構成が変わった場合は、そのフォーマットで有効な階級だけを表示する。

## 操作フロー

1. 登録済みユーザーが試合形式プルダウンで `match_format` を選ぶ。
2. 登録済みユーザーが階級プルダウンで `queue_name` を選ぶ。
3. 登録済みユーザーが参加ボタンを押す。
4. Bot は、選択された `match_format` と `queue_name` を使って `/join` と同じ業務処理を実行する。
5. 実行結果は、押下したユーザーにだけ見えるテキストメッセージで返す。

## 正常時の挙動

- 選択された `match_format` と `queue_name` の組み合わせでキュー参加を試みる。
- 返答は ephemeral なテキストメッセージで行う。
- 成功時の文言は以下とする。
  - `キューに参加しました。5分間マッチングします。`

## エラー時の挙動

- 試合形式が未選択:
  - ephemeral なテキストメッセージで `試合形式を選択してください。` を返す。
- 階級が未選択:
  - ephemeral なテキストメッセージで `階級を選択してください。` を返す。
- それ以外の業務エラー:
  - `/join` と同じ失敗文言を返す。
  - 例:
    - `指定したフォーマットは存在しません。`
    - `指定したキューは存在しません。`
    - `現在のレーティングではそのキューに参加できません。`
    - `現在キュー参加を制限されています。`
    - `すでにキュー参加中です。`
    - `キュー参加に失敗しました。管理者に確認してください。`

## 可視性

- 参加ボタン押下後の結果メッセージは、成功時も失敗時も押下したユーザーにだけ見えるようにする。
- `レート戦マッチング` チャンネルには、参加結果のたびに公開メッセージを追加送信しない。
- 常設メッセージ自体は、ユーザー操作ごとに更新しなくてよい。

## 冪等性

- 同一ユーザーが短時間に複数回参加ボタンを押した場合でも、キュー参加は 1 回だけ成立すればよい。
- すでに有効な待機キューを持つ状態で押された場合は、重複参加せず `すでにキュー参加中です。` を返す。

## 実装上の読み替え

- この UI は `/join` の入力方法だけを置き換えるものであり、キュー参加の業務ルール自体は変更しない。
- この UI が扱う入力は `match_format` と `queue_name` のみとする。
- 在席更新とキュー退出は別導線とし、現時点では slash command を正とする。
