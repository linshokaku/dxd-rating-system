# マッチ速報アナウンス UI 仕様

## UI 識別子

- `ui_type`: `matchmaking_news_match_announcement`

補足:

- この UI は `レート戦マッチ速報` チャンネルへ、マッチ成立ごとに動的生成するアナウンスメッセージとする。

## 目的

マッチ成立を登録済みユーザーへ公開し、観戦希望者に対して観戦応募導線を提供する。

## 対象外

- `レート戦マッチ速報` チャンネル自体の作成方法。
  - [setup_channel.md](setup_channel.md)
- 観戦応募の業務ルールそのもの。
  - [../matches/spectating.md](../matches/spectating.md)
- 試合参加者向け private thread UI。
  - [match_operation_thread.md](match_operation_thread.md)

## 前提

- Bot は、マッチ成立時に `レート戦マッチ速報` チャンネルへアナウンスメッセージを 1 件投稿する。
- アナウンスメッセージは `1 match_id` につき 1 件とする。
- このアナウンスは参加者向けの招集ではなく、観戦希望者向けの公開情報とする。
- 参加者への callout は private thread 側で行うため、この公開アナウンスでは参加者 mention を行わない。
- プレイヤー名の表示には、[../players/identity.md](../players/identity.md) の表示名方針に従って解決した表示名を使う。

## 表示要素

- メッセージには、少なくとも以下を含める。
  - マッチが成立したこと
  - 試合形式
  - 試合階級
  - Team A / Team B のチーム分け
  - 観戦希望者は button から応募すること
- Team A / Team B の参加者表示は、Discord の mention ではなく表示名テキストとする。
- `試合階級` は、対象 `queue_class_id` に対応するユーザー向けの `queue_name` を表示すればよい。
- メッセージには `観戦する` button を表示する。

## `観戦する` button

### 目的

公開アナウンスから、その試合への観戦応募を行えるようにする。

### 業務処理

- `観戦する` button は `/match_spectate` と同じ業務処理を実行する。
- 観戦応募に成功した場合は、そのユーザーを対応する試合運営 private thread に追加招待する。

### 正常時の挙動

- 押下したユーザーが登録済みであり、観戦受付中であり、参加者本人ではなく、枠が残っている場合に成功する。
- 押下直後には、押した本人にだけ見える ephemeral defer を返す。
- 成功時は、押した本人にだけ見える followup で以下を返す。
  - `観戦応募を受け付けました。現在 {active_spectator_count} / {max_spectators} 人です。`
- 成功時に、公開チャンネルへ追加の公開メッセージを送ることは必須としない。

### エラー時の挙動

- 登録前ユーザー:
  - `プレイヤー登録が必要です。先に /register を実行してください。`
- 観戦が制限されている:
  - `現在観戦を制限されています。`
- そのほかの試合状態エラー:
  - `/match_spectate` と同じ業務エラーを返す。
  - 例:
    - `指定した試合が見つかりません。`
    - `この試合は観戦受付を終了しています。`
    - `この試合の参加者は観戦応募できません。`
    - `この試合への観戦応募はすでに完了しています。`
    - `この試合の観戦枠は埋まっています。`
- 内部エラー:
  - `観戦応募に失敗しました。管理者に確認してください。`

## 可視性

- アナウンスメッセージ自体は `レート戦マッチ速報` チャンネルを閲覧できる登録済みユーザー全員に見える。
- button 押下結果は、成功時も失敗時も押した本人にだけ見える followup で返してよい。
- 観戦応募のたびに公開メッセージで人数更新を流し続けることは必須としない。

## 冪等性

- 同じユーザーが重複して押しても、観戦応募は 1 回だけ成立すればよい。
- 観戦受付終了後に古い button が押された場合は、既存メッセージを消さずに `この試合は観戦受付を終了しています。` を返せばよい。

## 既存コマンドとの対応

- `観戦する` -> `/match_spectate`
