# 登録 UI 仕様

## UI 識別子

- `ui_type`: `register_panel`

## 目的

未登録ユーザーが、Discord の特定の公開チャンネルに設置された UI からプレイヤー登録できるようにする。

## 対象外

- 登録 UI を設置する管理者コマンドの仕様。
- 利用規約のバージョン管理。
- 利用規約を本当に読んだことの保証。

## 前提

- 登録 UI は、コミュニティ内で案内用に使う特定の公開チャンネルに設置する。
- 登録 UI を置くチャンネル名の推奨値は `レート戦はこちらから` とする。
- 登録 UI の設置用チャンネルを作成するコマンド仕様は [setup_channel.md](setup_channel.md) を参照する。
- 登録ボタンを押すことを、利用規約への同意と同一視する。
- 登録時に、利用規約への別確認 checkbox や modal は出さない。
- 利用規約の同意履歴やバージョン別の再同意管理は、現時点では持たない。
- `/register` は引き続き有効な登録導線として残し、登録 UI とは同じ業務処理と文言を使う。

## 公開 UI の表示要素

- 登録案内用のメッセージを 1 つ設置する。
- メッセージには、少なくとも以下を表示する。
  - この UI がプレイヤー登録用であること。
  - ボタン押下をもって利用規約に同意したものとして扱うこと。
  - 登録後に Bot の各種機能を使えるようになること。
- メッセージには登録用 button を 1 つ設置する。
- 登録用 button のラベルは、利用規約への同意を含むことが利用者に伝わる文言にする。
- 現時点の推奨ラベルは `利用規約に同意して登録` とする。

## 操作フロー

1. 未登録ユーザーが登録 button を押す。
2. Bot は、その押下を利用規約への同意として扱い、押下ユーザーの Discord user ID をプレイヤー登録する。
3. 登録結果は、押下したユーザーにだけ見えるテキストメッセージで返す。

## 正常時の挙動

- 押下ユーザーの Discord user ID をプレイヤーとして登録する。
- 返答は ephemeral なテキストメッセージで行う。
- 成功時の文言は以下とする。
  - `登録が完了しました。`

## エラー時の挙動

- すでに登録済み:
  - ephemeral なテキストメッセージで `すでに登録済みです。` を返す。
- 内部エラー:
  - ephemeral なテキストメッセージで `登録に失敗しました。管理者に確認してください。` を返す。

## 可視性

- 登録 button 押下後の結果メッセージは、成功時も失敗時も押下したユーザーにだけ見えるようにする。
- 公開チャンネルには、登録成功や登録失敗の都度メッセージを追加送信しない。
- 公開チャンネル上の登録案内メッセージ自体は、押下のたびに内容を変更しなくてよい。

## 冪等性

- 同一ユーザーが短時間に複数回 button を押した場合でも、登録は 1 回だけ成立すればよい。
- すでに登録済みの状態で押された場合は、重複登録せず `すでに登録済みです。` を返す。

## 実装上の読み替え

- 登録 UI は `/register` の代替導線であり、プレイヤー登録の業務ルール自体は `/register` と同じものを使う。
- UI 固有の差分は、公開チャンネル上の常設メッセージから起動されることと、返答が ephemeral なテキストメッセージになることだけとする。
