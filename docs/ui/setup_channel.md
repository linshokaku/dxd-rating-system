# UI 設置チャンネル管理コマンド仕様

## 目的

管理者が、UI を設置するための専用チャンネルを個別または一括で作成し、必要に応じてそれらをまとめて撤収できるようにする。

## 対象

- UI の運用開始時に必要となる、専用チャンネルの個別作成、一括作成、初期 UI 配置。
- `/admin_setup_custom_ui_channel` または `/admin_setup_ui_channels` で作成した UI 設置チャンネルの一括撤収。

## 対象外

- 既存 UI チャンネルの再配置。
- 既存 UI メッセージの再生成。
- 利用中 UI の更新や差し替え。
- 個別 UI だけを選んで部分撤収する操作。

## 共通前提

- この Bot は 1 つの Discord サーバーでのみ運用する前提とする。
- この仕様でいうチャンネル作成、重複判定、削除対象判定は、その運用サーバー内だけを対象とする。
- 作成コマンドと撤収コマンドの実行結果は、成功時も失敗時も実行者にだけ見えるテキストメッセージで返す。
- UI の管理対象は、Bot が保持する UI 管理情報で判定する。
- チャンネル名だけを根拠に削除対象を決めない。

## 共通管理情報

- Bot は UI 管理用の内部情報として少なくとも以下を保持する。
  - `ui_type`
  - `channel_id`
  - `message_id`
  - `created_by_discord_user_id`
  - `created_at`

## `/admin_setup_custom_ui_channel`

### 目的

指定した UI を設置するための専用チャンネルを作成し、そのチャンネルに初期 UI を配置する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `ui_type` (必須): 設置したい UI の種別。
- `channel_name` (必須): 作成するチャンネル名。

初期状態で受け付ける `ui_type` は以下とする。

- `register_panel`

### 正常時の挙動

- 指定した `channel_name` で、指定した `ui_type` 用の公開テキストチャンネルを 1 つ作成する。
- 作成したチャンネルには、指定した `ui_type` に対応する初期 UI を 1 つ設置する。
- `ui_type=register_panel` の場合は、[register.md](register.md) で定義した登録 UI を設置する。
- チャンネル作成と UI 設置に成功したら、対応する UI 管理情報を保存する。
- 成功時のレスポンス文言は以下とする。
  - `UI 設置チャンネルを作成しました。`

### `register_panel` のチャンネル作成ルール

- 公開チャンネルとして作成する。
- 推奨チャンネル名は `レート戦はこちらから` とする。
- サーバーの一般ユーザーは、チャンネルを閲覧できる。
- サーバーの一般ユーザーは、設置済み button を押せる。
- サーバーの一般ユーザーには、新規メッセージ送信を許可しない。
- Bot と管理者は、UI 設置と保守に必要な権限を持つ前提とする。
- チャンネル内には、登録案内メッセージを 1 つ設置する。

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `ui_type` が不正な場合:
  - `指定した UI は存在しません。`
- `channel_name` が不正な場合:
  - `channel_name が不正です。`
- サーバー内に同名のチャンネルがすでに存在する場合:
  - `同名のチャンネルがすでに存在します。`
- 指定した `ui_type` が単一設置のみ許可される UI であり、すでに設置済みの場合:
  - `指定した UI はすでに設置済みです。`
- Bot にチャンネル作成またはメッセージ送信権限が不足している場合:
  - `Bot に必要な権限がありません。`
- 内部エラーが発生した場合:
  - `UI 設置チャンネルの作成に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 作成された UI チャンネルと、そこに配置された UI メッセージ自体は、対象ユーザーに公開される。

### 冪等性

- `register_panel` は、現時点ではこの Bot の運用サーバー内に 1 つだけ設置できる UI として扱う。
- すでに `register_panel` 用の管理対象チャンネルが存在する場合、追加作成は行わず `指定した UI はすでに設置済みです。` を返す。

### ロールバック

- チャンネル作成後に UI メッセージの設置に失敗した場合、Bot は作成したチャンネルの削除を試みる。
- ロールバックにも失敗した場合は、内部ログに残したうえで `UI 設置チャンネルの作成に失敗しました。管理者に確認してください。` を返す。

## `/admin_setup_ui_channels`

### 目的

レーティングシステムの運用開始に必要な UI 設置チャンネルを、推奨チャンネル名でまとめて作成し、必要な初期 UI を一括配置する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- なし。

### 作成対象

- このコマンドが作成対象とする「必要な全チャンネル」とは、この Bot の運用開始時に必要な UI 設置チャンネル全件を指す。
- 現時点の作成対象は以下とする。
  - `register_panel`: `レート戦はこちらから`
- 将来、運用開始時に必須となる UI が増えた場合は、この作成対象一覧に追加する。

### 正常時の挙動

- Bot は作成対象一覧を参照し、未設置の UI だけを作成対象として抽出する。
- 各作成対象について、対応する推奨チャンネル名で公開テキストチャンネルを 1 つ作成する。
- 作成した各チャンネルには、対応する初期 UI を 1 つ設置する。
- 各チャンネル作成と UI 設置に成功したら、対応する UI 管理情報を保存する。
- 1 件以上のチャンネルを新規作成できた場合、成功時のレスポンス文言は以下とする。
  - `必要な UI 設置チャンネルを作成しました。`

### すべて設置済みの場合の挙動

- 作成対象の UI がすべて設置済みである場合は、エラーにはしない。
- この場合のレスポンス文言は以下とする。
  - `必要な UI 設置チャンネルはすでに作成済みです。`

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- 推奨チャンネル名と同名のチャンネルがサーバー内にすでに存在し、かつそのチャンネルが Bot の管理対象ではない場合:
  - `推奨チャンネル名のチャンネルがすでに存在します。`
- Bot にチャンネル作成またはメッセージ送信権限が不足している場合:
  - `Bot に必要な権限がありません。`
- 内部エラーが発生した場合:
  - `必要な UI 設置チャンネルの作成に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 作成された UI チャンネルと、そこに配置された UI メッセージ自体は、対象ユーザーに公開される。

### 整合性

- すでに設置済みの UI は作成対象から除外し、未設置の UI だけを新規作成する。
- 同名チャンネルの衝突判定は、未設置 UI の推奨チャンネル名に対して行う。
- 一括作成中に一部の UI 作成だけが成功した状態をできるだけ残さないようにする。

### 冪等性

- すでに一部の必要 UI が設置済みの状態で再実行しても、安全に未設置分だけを補完できる。
- すでにすべての必要 UI が設置済みの状態で再実行した場合は `必要な UI 設置チャンネルはすでに作成済みです。` を返す。

### ロールバック

- 一括作成中にチャンネル作成または UI 設置で失敗した場合、Bot はその実行で新規に作成したチャンネルだけを削除することを試みる。
- ロールバックに成功した場合、失敗した実行で新規に保存した UI 管理情報も削除する。
- ロールバックにも失敗した場合は、内部ログに残したうえで `必要な UI 設置チャンネルの作成に失敗しました。管理者に確認してください。` を返す。

## `/admin_teardown_ui_channels`

### 目的

`/admin_setup_custom_ui_channel` または `/admin_setup_ui_channels` で作成した UI 設置チャンネルを一括で削除し、Bot の UI 運用を撤収する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `confirm` (必須): 撤収実行の確認入力。

受け付ける値は以下のみとする。

- `teardown`

### 正常時の挙動

- Bot が保持する UI 管理情報を参照し、管理対象の UI 設置チャンネルをすべて列挙する。
- 列挙したチャンネルをすべて削除対象とする。
- 各チャンネルの削除に成功したら、対応する UI 管理情報も削除する。
- すべての削除処理が成功した場合、成功レスポンスを返す。
- 成功時のレスポンス文言は以下とする。
  - `UI 設置チャンネルをすべて撤収しました。`

### 対象チャンネルがない場合の挙動

- 管理対象の UI 設置チャンネルが 1 つも存在しない場合でも、エラーにはしない。
- この場合のレスポンス文言は以下とする。
  - `撤収対象の UI 設置チャンネルはありません。`

### 削除ルール

- 削除対象は、Bot が管理情報として保持しているチャンネル ID に一致するものだけとする。
- 管理対象チャンネルに紐づく UI メッセージは、チャンネル削除に伴ってまとめて削除される前提とする。
- すでに手動削除されて存在しないチャンネルが管理情報に残っていた場合は、そのチャンネルは削除済みとして扱い、残っている管理情報だけを削除して処理を継続する。
- 一部のチャンネルだけが削除できなかった場合は、成功扱いにしない。

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `confirm` が不正な場合:
  - `confirm が不正です。`
- Bot にチャンネル削除権限が不足している場合:
  - `Bot に必要な権限がありません。`
- 一部または全部のチャンネル削除に失敗した場合:
  - `UI 設置チャンネルの撤収に失敗しました。管理者に確認してください。`
- 内部エラーが発生した場合:
  - `UI 設置チャンネルの撤収に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 撤収された UI チャンネルは公開状態から消える。

### 整合性

- 管理情報の削除は、対応するチャンネルの削除成功後に行う。
- すでに存在しないチャンネルについては、実体削除は不要なので、管理情報の削除をもって整合性回復とみなす。
- 一部削除に失敗した場合は、削除済みチャンネル分の結果は保持し、未削除チャンネルの管理情報は残す。

### 冪等性

- すでにすべての UI 設置チャンネルが撤収済みの状態で再実行しても安全に実行できる。
- この場合は `撤収対象の UI 設置チャンネルはありません。` を返す。

## 拡張方針

- これらのコマンドは `ui_type` ごとに、以下の設定を切り替えられる前提で設計する。
  - 作成するチャンネル種別
  - 推奨チャンネル名
  - 既定の permission overwrite
  - 初期配置する UI メッセージ内容
  - 単一設置か複数設置可能か
- 一括作成コマンドは、運用開始時に必要な `ui_type` と推奨チャンネル名の一覧を参照して作成対象を決定できるようにする。
- 新しい UI を追加する場合は、作成コマンドの `ui_type` の選択肢を拡張し、対応するチャンネル作成ルールと初期 UI 配置処理を追加する。
- 将来 UI 種別が増えても、管理対象として登録されている UI チャンネルであれば撤収コマンドでまとめて撤収できるようにする。
