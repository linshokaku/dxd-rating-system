# UI 設置チャンネル管理コマンド仕様

## 目的

管理者が、UI を設置するための専用チャンネルを個別または一括で作成し、必要に応じて既存 managed UI チャンネルの再セットアップや一括撤収を行えるようにする。

## 対象

- UI の運用開始時に必要となる、専用チャンネルの個別作成、一括作成、初期 UI 配置。
- 既存 managed UI チャンネル 1 件の再セットアップ。
- `/admin_setup_ui_channels` の実行を妨げる重複名チャンネルの cleanup。
- `/admin_setup_custom_ui_channel` または `/admin_setup_ui_channels` で作成した UI 設置チャンネルの一括撤収。

## 対象外

- 既存 UI チャンネルの再配置やカテゴリ移動だけを行う操作。
- 既存 UI メッセージだけを個別に再生成する操作。
- 利用中 UI の更新や差し替え。
- 個別 UI だけを選んで部分撤収する操作。
- Bot UI を設置しない運用チャンネルの作成や削除。

## 共通前提

- この Bot は 1 つの Discord サーバーでのみ運用する前提とする。
- この仕様でいうチャンネル作成、重複判定、削除対象判定は、その運用サーバー内だけを対象とする。
- 作成、再セットアップ、cleanup、撤収コマンドの実行結果は、成功時も失敗時も実行者にだけ見えるテキストメッセージで返す。
- UI の管理対象は、Bot が保持する UI 管理情報で判定する。
- チャンネル名だけを根拠に削除対象を決めない。
- 登録済みユーザー向けチャンネル全体の用途と権限は [registered_channels.md](registered_channels.md) を参照する。
- Bot に必要な Discord 権限全体は [../discord_permissions.md](../discord_permissions.md) を参照する。

## 共通管理情報

- Bot は UI 管理用の内部情報として少なくとも以下を保持する。
  - `ui_type`
  - `channel_id`
  - `message_id`
  - `status_message_id`
  - `matchmaking_one_v_one_message_id`
  - `matchmaking_two_v_two_message_id`
  - `matchmaking_three_v_three_message_id`
  - `created_by_discord_user_id`
  - `created_at`
- `message_id` は、初期メッセージが 1 通だけの UI に対して使う。
- `register_panel`、`matchmaking_news_channel`、`info_channel`、`system_announcements_channel`、`admin_contact_channel`、`admin_operations_channel` では `message_id` に初期メッセージ ID を保存する。
- `matchmaking_channel` では `message_id` は使わず、`status_message_id` と 3 つの形式別 panel message ID で初期 UI を管理する。

## `/admin_setup_custom_ui_channel`

### 目的

指定した UI を設置するための専用チャンネルを作成し、そのチャンネルに初期 UI を配置する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `ui_type` (必須): 設置したい UI の種別。
- `channel_name` (必須): 作成するチャンネル名。

初期状態で受け付ける `ui_type` は以下とする。

- `register_panel`
- `matchmaking_channel`
- `matchmaking_news_channel`
- `info_channel`
- `system_announcements_channel`
- `admin_contact_channel`
- `admin_operations_channel`

### 正常時の挙動

- 指定した `channel_name` で、指定した `ui_type` 用のテキストチャンネルを 1 つ作成する。
- 作成したチャンネルには、指定した `ui_type` に対応する初期メッセージまたは初期 UI を設置する。
- `ui_type=register_panel` の場合は、[register.md](register.md) で定義した登録 UI を設置する。
- `ui_type=matchmaking_channel` の場合は、[matchmaking_channel.md](matchmaking_channel.md) で定義したマッチング UI を設置する。
- `ui_type=info_channel` の場合は、[info_channel.md](info_channel.md) で定義した情報確認導線 UI を設置する。
- チャンネル作成と UI 設置に成功したら、対応する UI 管理情報を保存する。
- 成功時のレスポンス文言は以下とする。
  - `UI 設置チャンネルを作成しました。`

### `register_panel` のチャンネル作成ルール

- 公開チャンネルとして作成する。
- 推奨チャンネル名は `レート戦はこちらから` とする。
- サーバーの一般ユーザーは、チャンネルを閲覧できる。
- サーバーの一般ユーザーは、設置済み UI を操作できる。
- サーバーの一般ユーザーには、新規メッセージ送信を許可しない。
- Bot と管理者は、UI 設置と保守に必要な権限を持つ前提とする。
- チャンネル内には、登録案内メッセージを 1 つ設置する。

### `matchmaking_channel` のチャンネル作成ルール

- 公開テキストチャンネルとして作成する。
- 推奨チャンネル名は `レート戦マッチング` とする。
- 登録済みユーザー向け role を持つユーザーだけが閲覧できる。
- 一般ユーザーには、新規メッセージ送信を許可しない。
- 一般ユーザーには、public thread 作成を許可しない。
- 一般ユーザーには、private thread 作成を許可しない。
- チャンネル内には、以下の 5 通の初期メッセージを順に設置する。
  1. 遊び方メッセージ
  2. 参加状況メッセージ
  3. `1v1` 用の参加 panel
  4. `2v2` 用の参加 panel
  5. `3v3` 用の参加 panel
- 2 通目の参加状況メッセージには `更新する` ボタンを設置する。
- 3 通目から 5 通目の参加 panel は、それぞれ対応する形式専用の queue 選択 UI と参加ボタンで構成する。
- `matchmaking_channel` の UI 管理情報は、`status_message_id` に 2 通目の message ID を保存し、3 通目から 5 通目の message ID は `matchmaking_one_v_one_message_id`、`matchmaking_two_v_two_message_id`、`matchmaking_three_v_three_message_id` に保存する。
- `matchmaking_channel` では `message_id` を使わない。
- Bot は、運用上必要な private thread 作成とメッセージ送信を行える前提とする。

### `matchmaking_news_channel` のチャンネル作成ルール

- 公開テキストチャンネルとして作成する。
- 推奨チャンネル名は `レート戦マッチ速報` とする。
- 登録済みユーザー向け role を持つユーザーだけが閲覧できる。
- 一般ユーザーには、新規メッセージ送信を許可しない。
- 一般ユーザーには、public thread 作成を許可しない。
- 一般ユーザーには、private thread 作成を許可しない。
- チャンネル内には、用途説明用の初期メッセージを 1 つ設置する。

### `info_channel` のチャンネル作成ルール

- 公開テキストチャンネルとして作成する。
- 推奨チャンネル名は `レート戦情報` とする。
- 登録済みユーザー向け role を持つユーザーだけが閲覧できる。
- 一般ユーザーには、新規メッセージ送信を許可しない。
- 一般ユーザーには、public thread 作成を許可しない。
- 一般ユーザーには、private thread 作成を許可しない。
- チャンネル内には、情報確認導線用の初期メッセージを 1 つ設置する。
- 初期メッセージには、4 種類の `/info_thread command_name:<literal>` と同等の操作が行える button UI を含める。
- この公開チャンネル上の UI は info thread 作成導線であり、作成後の private thread 内に将来置く command-specific な button / pulldown UI とは別物として扱う。
- Bot は、運用上必要な private thread 作成と thread 内メッセージ送信を行える前提とする。
- 公開 button UI の詳細仕様は [info_channel.md](info_channel.md) を参照する。

### `system_announcements_channel` のチャンネル作成ルール

- 公開テキストチャンネルとして作成する。
- 推奨チャンネル名は `レート戦アナウンス` とする。
- 登録済みユーザー向け role を持つユーザーだけが閲覧できる。
- 一般ユーザーには、新規メッセージ送信を許可しない。
- 一般ユーザーには、public thread 作成を許可しない。
- 一般ユーザーには、private thread 作成を許可しない。
- チャンネル内には、用途説明用の初期メッセージを 1 つ設置する。

### `admin_contact_channel` のチャンネル作成ルール

- 公開テキストチャンネルとして作成する。
- 推奨チャンネル名は `運営連絡・フィードバック` とする。
- 誰でも閲覧できる。
- 誰でも通常メッセージを送信できる。
- チャンネル内には、用途説明用の初期メッセージを 1 つ設置する。

### `admin_operations_channel` のチャンネル作成ルール

- private text channel として作成する。
- 推奨チャンネル名は `運営専用` とする。
- `@everyone` には閲覧権限を付与しない。
- 登録済みユーザー向け role にも閲覧権限を付与しない。
- `SUPER_ADMIN_USER_IDS` に含まれる Discord user ID に対応するサーバーメンバーだけが閲覧できる。
- super admin は通常メッセージを送信できる。
- Bot は運用通知投稿と保守上必要なメッセージ送信を行える前提とする。
- thread 作成は初期スコープ外とし、通常メッセージだけを前提にする。
- チャンネル内には、用途説明用の初期メッセージを 1 つ設置する。
- 詳細仕様は [admin_operations_channel.md](admin_operations_channel.md) を参照する。

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `ui_type` が不正な場合:
  - `指定した UI は存在しません。`
- `channel_name` が不正な場合:
  - `channel_name が不正です。`
- サーバー内に同名のチャンネルがすでに存在する場合:
  - `同名のチャンネルがすでに存在します。`
- 指定した `ui_type` が単一設置のみ許可される UI であり、すでに設置済みの場合:
  - `指定した UI はすでに設置済みです。`
- Bot にチャンネル作成またはメッセージ送信権限が不足している場合:
  - `Bot に必要な権限がありません。`
  - 実装で判定できる場合は、`不足している権限: ...` を続けて表示してよい。
  - Discord API が返した `403 Forbidden` の詳細を続けて表示してよい。
- 内部エラーが発生した場合:
  - `UI 設置チャンネルの作成に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 作成された UI チャンネルと、そこに配置された UI メッセージ自体は、対象ユーザーに公開される。

### 冪等性

- `register_panel` は、現時点ではこの Bot の運用サーバー内に 1 つだけ設置できる UI として扱う。
- すでに `register_panel` 用の管理対象チャンネルが存在する場合、追加作成は行わず `指定した UI はすでに設置済みです。` を返す。

### ロールバック

- チャンネル作成後に UI メッセージの設置に失敗した場合、Bot は作成したチャンネルの削除を試みる。
- ロールバックにも失敗した場合は、内部ログに残したうえで `UI 設置チャンネルの作成に失敗しました。管理者に確認してください。` を返す。

## `/admin_setup_ui_channels`

### 目的

レーティングシステムの運用開始に必要な UI 設置チャンネルを、推奨チャンネル名でまとめて作成し、必要な初期 UI を一括配置する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- なし。

### 作成対象

- このコマンドが作成対象とする「必要な全チャンネル」とは、この Bot の運用開始時に必要な UI 設置チャンネル全件を指す。
- 現時点の作成対象は以下とする。
  - `register_panel`: `レート戦はこちらから`
  - `matchmaking_channel`: `レート戦マッチング`
  - `matchmaking_news_channel`: `レート戦マッチ速報`
  - `info_channel`: `レート戦情報`
  - `system_announcements_channel`: `レート戦アナウンス`
  - `admin_contact_channel`: `運営連絡・フィードバック`
  - `admin_operations_channel`: `運営専用`
- 将来、運用開始時に必須となる UI が増えた場合は、この作成対象一覧に追加する。

### 正常時の挙動

- Bot は作成対象一覧を参照し、未設置の UI だけを作成対象として抽出する。
- 各作成対象について、対応する推奨チャンネル名と権限設定でテキストチャンネルを 1 つ作成する。
- 作成した各チャンネルには、対応する初期メッセージまたは初期 UI を設置する。
- 各チャンネル作成と UI 設置に成功したら、対応する UI 管理情報を保存する。
- 1 件以上のチャンネルを新規作成できた場合、成功時のレスポンス文言は以下とする。
  - `必要な UI 設置チャンネルを作成しました。`

### すべて設置済みの場合の挙動

- 作成対象の UI がすべて設置済みである場合は、エラーにはしない。
- この場合のレスポンス文言は以下とする。
  - `必要な UI 設置チャンネルはすでに作成済みです。`

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- 推奨チャンネル名と同名のチャンネルがサーバー内にすでに存在し、かつそのチャンネルが Bot の管理対象ではない場合:
  - `推奨チャンネル名のチャンネルがすでに存在します。`
  - 必要に応じて `/admin_cleanup_ui_channels` で障害となる重複名チャンネルを削除してから再実行してよい。
- Bot にチャンネル作成またはメッセージ送信権限が不足している場合:
  - `Bot に必要な権限がありません。`
  - 実装で判定できる場合は、`不足している権限: ...` を続けて表示してよい。
  - Discord API が返した `403 Forbidden` の詳細を続けて表示してよい。
- 内部エラーが発生した場合:
  - `必要な UI 設置チャンネルの作成に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 作成された UI チャンネルと、そこに配置された UI メッセージ自体は、対象ユーザーにだけ公開される。

### 整合性

- すでに設置済みの UI は作成対象から除外し、未設置の UI だけを新規作成する。
- 同名チャンネルの衝突判定は、未設置 UI の推奨チャンネル名に対して行う。
- 一括作成中に一部の UI 作成だけが成功した状態をできるだけ残さないようにする。

### 冪等性

- すでに一部の必要 UI が設置済みの状態で再実行しても、安全に未設置分だけを補完できる。
- すでにすべての必要 UI が設置済みの状態で再実行した場合は `必要な UI 設置チャンネルはすでに作成済みです。` を返す。

### ロールバック

- 一括作成中にチャンネル作成または UI 設置で失敗した場合、Bot はその実行で新規に作成したチャンネルだけを削除することを試みる。
- ロールバックに成功した場合、失敗した実行で新規に保存した UI 管理情報も削除する。
- ロールバックにも失敗した場合は、内部ログに残したうえで `必要な UI 設置チャンネルの作成に失敗しました。管理者に確認してください。` を返す。

## `/admin_resetup_ui_channel`

### 目的

すでに Bot が管理している UI チャンネル 1 件を選択し、その `ui_type` に対応する初期状態へ再セットアップする。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `ui_type` (必須): 再セットアップしたい UI の種別。
- `confirm` (必須): 再セットアップ実行の確認入力。

初期状態で受け付ける `ui_type` は以下とする。

- `register_panel`
- `matchmaking_channel`
- `matchmaking_news_channel`
- `info_channel`
- `system_announcements_channel`
- `admin_contact_channel`
- `admin_operations_channel`

`confirm` に受け付ける値は以下のみとする。

- `resetup`

### 正常時の挙動

- Bot は `ui_type` から管理対象の UI 管理情報を 1 件解決する。
- 管理対象の元チャンネルが運用サーバー内に実在することを必須とする。
- Bot は元チャンネルの現在名を引き継いだ replacement channel を、新規 setup 時と同じ `ui_type` 別ルール、権限設定、固定カテゴリ運用で作成する。
- replacement channel には、その `ui_type` に対応する初期メッセージまたは初期 UI を新規に配置する。
- replacement channel の準備が完了してから、UI 管理情報を新しい `channel_id` と message id 群へ差し替える。
- UI 管理情報の差し替え後、Bot は旧チャンネルを削除する。
- `matchmaking_channel` を再セットアップする場合は、旧チャンネルの 2 通目から 5 通目に相当する managed UI message は再利用せず、replacement channel 上で新しい `status_message_id` と形式別 panel message ID を採番し直す。
- 成功時のレスポンス文言は以下とする。
  - `指定した UI チャンネルを再セットアップしました。`

### 再セットアップ時の削除ルール

- このコマンドは destructive な操作として扱う。
- 旧チャンネル配下の履歴、通常メッセージ、public/private thread、既存 UI message は旧チャンネル削除とともに失われる前提とする。
- 旧チャンネル上のメッセージや thread を replacement channel に引き継がない。
- 既存 UI message だけを残して新しい UI を追加投稿することは行わない。

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `ui_type` が不正な場合:
  - `指定した UI は存在しません。`
- `confirm` が不正な場合:
  - `confirm が不正です。`
- 指定した `ui_type` が未設置の場合:
  - `指定した UI は未設置です。`
- 指定した `ui_type` の管理対象チャンネルが見つからない場合:
  - `指定した UI の管理対象チャンネルが見つかりません。`
- Bot にチャンネル作成、チャンネル削除、またはメッセージ送信権限が不足している場合:
  - `Bot に必要な権限がありません。`
  - 実装で判定できる場合は、`不足している権限: ...` を続けて表示してよい。
  - Discord API が返した `403 Forbidden` の詳細を続けて表示してよい。
- 内部エラーが発生した場合:
  - `UI 設置チャンネルの再セットアップに失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 実行結果は、成功時も失敗時も実行者にだけ見えるテキストメッセージで返す。
- replacement channel とそこに配置された初期 UI は、対象ユーザーに公開される。

### 整合性

- 再セットアップ対象は、Bot が UI 管理情報として保持している `ui_type` 1 件に限定する。
- replacement channel の作成と初期 UI 配置が完了するまで、既存 UI 管理情報は差し替えない。
- 同じ `ui_type` に対して管理対象チャンネルが複数存在する状態は、このコマンドの正常完了後に残さない。

### 冪等性

- 同じ `ui_type` に対して再実行した場合でも、最終的には指定した `ui_type` の managed UI channel が 1 件だけ残る状態へ収束させる。
- ただし、このコマンドは毎回新しい replacement channel を作るため、Discord 上の `channel_id` と managed UI message ID は再実行のたびに変わってよい。
- 再実行のたびに、その時点の旧チャンネル配下の履歴と thread は失われる。

### ロールバック

- replacement channel の作成、初期 UI 配置、または UI 管理情報の差し替えに失敗した場合、Bot は新規に作成した replacement channel の削除を試み、元チャンネルと元の UI 管理情報を維持する。
- 旧チャンネルの削除に失敗した場合も、Bot は replacement channel の削除と UI 管理情報の復元を試みる。
- ロールバックにも失敗した場合は、内部ログに残したうえで `UI 設置チャンネルの再セットアップに失敗しました。管理者に確認してください。` を返す。

## `/admin_cleanup_ui_channels`

### 目的

`/admin_setup_ui_channels` の実行を妨げる、推奨チャンネル名と重複している未管理チャンネルを削除する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `confirm` (必須): cleanup 実行の確認入力。

受け付ける値は以下のみとする。

- `cleanup`

### 削除対象

- 削除対象は、`/admin_setup_ui_channels` がこれから作成しようとする必須 UI チャンネルの推奨チャンネル名と一致するチャンネルのうち、Bot の UI 管理情報に含まれていないものだけとする。
- すでに管理対象として登録済みの UI に対応するチャンネル名重複は、このコマンドの削除対象に含めない。
- Bot の管理対象チャンネルそのものは、このコマンドでは削除しない。

### 正常時の挙動

- Bot は UI 管理情報を参照し、未設置の必須 UI を特定する。
- 未設置 UI の推奨チャンネル名と一致する未管理チャンネルを、cleanup 対象として列挙する。
- 列挙したチャンネルをすべて削除する。
- すべての削除処理が成功した場合、成功レスポンスを返す。
- 成功時のレスポンス文言は以下とする。
  - `setup の障害となる重複チャンネルを削除しました。`

### 削除対象がない場合の挙動

- cleanup 対象の重複チャンネルが 1 つも存在しない場合でも、エラーにはしない。
- この場合のレスポンス文言は以下とする。
  - `削除対象の重複チャンネルはありません。`

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `confirm` が不正な場合:
  - `confirm が不正です。`
- Bot にチャンネル削除権限が不足している場合:
  - `Bot に必要な権限がありません。`
  - 実装で判定できる場合は、`不足している権限: ...` を続けて表示してよい。
  - Discord API が返した `403 Forbidden` の詳細を続けて表示してよい。
- 一部または全部のチャンネル削除に失敗した場合:
  - `重複チャンネルの cleanup に失敗しました。管理者に確認してください。`
- 内部エラーが発生した場合:
  - `重複チャンネルの cleanup に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- cleanup で削除されたチャンネルは公開状態から消える。

### 冪等性

- すでに cleanup 対象が存在しない状態で再実行しても安全に実行できる。
- この場合は `削除対象の重複チャンネルはありません。` を返す。

## `/admin_teardown_ui_channels`

### 目的

`/admin_setup_custom_ui_channel` または `/admin_setup_ui_channels` で作成した UI 設置チャンネルを一括で削除し、Bot の UI 運用を撤収する。

### 実行者

- `admin` 権限を持つユーザーのみ。

### 入力

- `confirm` (必須): 撤収実行の確認入力。

受け付ける値は以下のみとする。

- `teardown`

### 正常時の挙動

- Bot が保持する UI 管理情報を参照し、管理対象の UI 設置チャンネルをすべて列挙する。
- 列挙したチャンネルをすべて削除対象とする。
- 各チャンネルの削除に成功したら、対応する UI 管理情報も削除する。
- すべての削除処理が成功した場合、成功レスポンスを返す。
- 成功時のレスポンス文言は以下とする。
  - `UI 設置チャンネルをすべて撤収しました。`

### 対象チャンネルがない場合の挙動

- 管理対象の UI 設置チャンネルが 1 つも存在しない場合でも、エラーにはしない。
- この場合のレスポンス文言は以下とする。
  - `撤収対象の UI 設置チャンネルはありません。`

### 削除ルール

- 削除対象は、Bot が管理情報として保持しているチャンネル ID に一致するものだけとする。
- 管理対象チャンネルに紐づく UI メッセージは、チャンネル削除に伴ってまとめて削除される前提とする。
- すでに手動削除されて存在しないチャンネルが管理情報に残っていた場合は、そのチャンネルは削除済みとして扱い、残っている管理情報だけを削除して処理を継続する。
- 一部のチャンネルだけが削除できなかった場合は、成功扱いにしない。

### エラー時の挙動

- 実行者に `admin` 権限がない場合:
  - `このコマンドは管理者のみ実行できます。`
- `confirm` が不正な場合:
  - `confirm が不正です。`
- Bot にチャンネル削除権限が不足している場合:
  - `Bot に必要な権限がありません。`
  - 実装で判定できる場合は、`不足している権限: ...` を続けて表示してよい。
  - Discord API が返した `403 Forbidden` の詳細を続けて表示してよい。
- 一部または全部のチャンネル削除に失敗した場合:
  - `UI 設置チャンネルの撤収に失敗しました。管理者に確認してください。`
- 内部エラーが発生した場合:
  - `UI 設置チャンネルの撤収に失敗しました。管理者に確認してください。`
  - 内部例外の詳細は Discord に出さない。

### 可視性

- 撤収された UI チャンネルは公開状態から消える。

### 整合性

- 管理情報の削除は、対応するチャンネルの削除成功後に行う。
- すでに存在しないチャンネルについては、実体削除は不要なので、管理情報の削除をもって整合性回復とみなす。
- 一部削除に失敗した場合は、削除済みチャンネル分の結果は保持し、未削除チャンネルの管理情報は残す。

### 冪等性

- すでにすべての UI 設置チャンネルが撤収済みの状態で再実行しても安全に実行できる。
- この場合は `撤収対象の UI 設置チャンネルはありません。` を返す。

## 拡張方針

- これらのコマンドは `ui_type` ごとに、以下の設定を切り替えられる前提で設計する。
  - 作成するチャンネル種別
  - 推奨チャンネル名
  - 既定の permission overwrite
  - 初期配置する UI メッセージ内容
  - 単一設置か複数設置可能か
- 一括作成コマンドは、運用開始時に必要な `ui_type` と推奨チャンネル名の一覧を参照して作成対象を決定できるようにする。
- 新しい UI を追加する場合は、作成コマンドの `ui_type` の選択肢を拡張し、対応するチャンネル作成ルールと初期 UI 配置処理を追加する。
- `matchmaking_channel`、`matchmaking_news_channel`、`info_channel` のような追加 UI を定義した場合も、このコマンド群で管理対象に取り込めるようにする。
- 将来 UI 種別が増えても、管理対象として登録されている UI チャンネルであれば撤収コマンドでまとめて撤収できるようにする。
